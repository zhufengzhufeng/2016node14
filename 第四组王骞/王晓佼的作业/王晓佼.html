git 分支

  • 根据主干建立分支，当代码确认无误后，合并到主干上

  • 分支的创建

    • git branch –》查看分支

    • git branch+分支名字 –>创建分支

      • git branch 查看新分支是否增加成功

  • 分支的删除

    • git branch -D+分支名字

  • 分支的创建和删除

    • git checkout -b(分支)+分支名
  • 分支的管理合并
    • 内容提交才归分支管理，否则不能管理
    • git merge+分支名 –》合并分支（合并后分支还在，只是经过合并，分支的内容一样）
  • 分支主干综合
    • 分支和主干一样，主干{div：color：red} ,分支{div{color:blue}}
      • 创建分支，编辑分支，推送分支，checkout master ,(git commit -a -m”master.js 首次提交不可以使用这种方式)，合并（失败：有冲突删掉），不要的文件删除，再次提交并合并
    • 冲突的解决
      • 在文件夹保存想要的（去掉>>>>><<<<<====）
      • 提交最终结果

git远程

  • 发布
    • 线上有内容：先拿下来，再提交
    • 初始化仓库
  • git push xxx(要推送的文件名) master
  • 本地同步到远程流程
    • git status(查看状态)–》建立忽略文件（.gitignore手动添加忽略 【.idea 忽略的文件】）–》【绿色是可提交状态】git commit -m”名字’–》git remote add +名字–》github创建仓库（Reposibility ）+地址–》git push origin master -u
  • 创建readMe文件
    • echo ‘# welcome ’ >>
  • 添加忽略文件
    • touch .gitignore （ .DS_Store 苹果） .idea（手动在gitignore中添加）
  • 添加远程仓库
    • git remote add origin https://github.com/wangxiaojiao2016/test.git
  • 删除远程仓库
    • git remote rm+仓库名
  • 推送到远程仓库
    • git push origin master -u(upstream) 【写了-u 下次直接 git push就可以】
